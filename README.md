# Nexus Sign.AI: Real-Time Sign Language Recognition System

Nexus Sign.AI is an advanced, real-time sign language to text translation system. Designed for high performance and low latency, it processes video streams to detect, track, and classify complex hand gestures into human-readable text.

## Core Architecture

The system is built on a high-throughput pipeline that leverages edge-computing capabilities within the browser, ensuring privacy and zero-latency inference. 

### 1. Vision Core: MediaPipe Hand Landmarker
At the heart of the detection engine is Google's **MediaPipe Vision Tasks**. 
- We utilize the `@mediapipe/tasks-vision` package, running via WebAssembly (WASM) to achieve near-native execution speeds.
- The system instantiates a `HandLandmarker` model optimized with `float16` precision and GPU delegation.
- For every frame of the video feed, the Vision Core detects up to two hands simultaneously, extracting **21 3D spatial landmarks** (x, y, z coordinates) per hand. This topological mapping forms the skeletal structure required for gesture analysis.

### 2. Gesture Recognition Engine
The spatial coordinates generated by the Vision Core are passed to a highly specialized, custom-trained classification model. 
- **Target Dictionary:** The model has been meticulously trained on a specific vocabulary dataset (including signs like *"WE"*, *"SIGN"*, *"FOR"*, *"HUMAN"*, *"RIGHTS"*, *"LOLA"*).
- **Coordinate Normalization:** Before inference, the raw `(x, y, z)` landmark coordinates are normalized relative to the wrist node (`node 0`) and the hand's bounding box. This ensures the model is invariant to the subject's distance from the camera or their position within the frame.
- **Temporal Analysis:** Recognizing sign language requires understanding motion over time. The engine utilizes a sliding temporal window, analyzing the trajectory and velocity of the nodes across sequential frames rather than relying on a single static image. 

### 3. Stream Processing Pipeline
The application orchestrates the data flow using a highly optimized `requestAnimationFrame` loop synced directly to the video playback rate:
1. **Frame Capture:** The `<video>` element acts as the source tensor.
2. **Landmark Extraction:** `handLandmarker.detectForVideo()` processes the current timestamp.
3. **Skeleton Rendering:** A `<canvas>` layer immediately renders the topological connections (thumb, index, middle, ring, pinky vectors) overlaying the feed for visual feedback.
4. **Classification:** The normalized landmark vectors are classified by the gesture engine.
5. **State Aggregation:** If a gesture sustains a high confidence threshold (typically >85%) over consecutive frames, it is committed to the global state and appended to the `RAW_DETECTION_STREAM`.

## Performance Considerations

- **Asynchronous Initialization:** The WASM binary and model weights (`hand_landmarker.task`) are fetched and initialized asynchronously during the application load phase, ensuring the UI remains responsive.
- **GPU Acceleration:** By configuring `delegate: "GPU"`, the heavy matrix multiplications required for spatial feature extraction are offloaded from the CPU.
- **Garbage Collection:** The inference loop is strictly optimized to reuse memory buffers where possible, preventing GC (Garbage Collection) pauses that would otherwise cause frame drops during live translation.

## Technology Stack
* **Framework:** React 19 / Vite
* **Computer Vision:** `@mediapipe/tasks-vision` (WASM + WebGL)
* **Styling & Animation:** TailwindCSS v4, Framer Motion
* **State Management:** React Hooks with optimized re-render boundaries
